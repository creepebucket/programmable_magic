# Repository Guidelines

1. 本项目基于 neoforge 1.21.8，任何实现都必须与该版本保持一致。
2. **始终使用中文** 进行思考与回复。
3. 查文档后再写 API，绝不胡乱猜测。
4. 禁用所有过时或旧 API，它们不是可替代方案。
5. 坚持 KISS 原则，方案必须简单直接。
6. 任何重复的逻辑或方法都要抽到父类，避免在子类里堆叠。
7. 排查 bug 必须定位根本原因，修复源头，而不是随便打补丁。
8. 禁写防御性或“兜底”代码，那样既不解决问题也增加调试成本。
9. 不得改动我已有的注释内容。
10. 不得把本文件的内容抄进代码注释。
11. 如果修 bug 无法在 3 行以内完成，就不要直接改代码，只需说明原因并给出建议。
12. 修 bug 时要顺着我现在的设计逻辑来，别动不动就推翻或重写整体逻辑，先按现有实现思路去定位并解决。
13. 根目录已有文档，动手前先阅读。
14. 代码保持平铺式：按执行顺序直接写在当前上下文，流程清晰、调用栈浅。
15. 所有表达式折叠、括号求值等都走既有工具（SpellUtils 等），不要复制粘贴新的求值链。
16. 默认信任我构造的状态，必要检查提前放在固定位置，别到处加防御性判断或兜底逻辑。
17. 在字符串中, 使用snake_case 命名法而不是 camelCase。
