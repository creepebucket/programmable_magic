# 项目概述

## 🎯 项目简介

可编程魔法模组是一个基于 NeoForge 1.21.1 的 Minecraft 模组，它将编程的概念融入到魔法系统中，让玩家能够像编写程序一样组合法术来创造各种魔法效果。

## 🌟 核心特色

### 1. 可编程的魔法系统
- **法术组合**：就像编程语言中的函数调用
- **参数传递**：支持不同类型的数据作为法术参数
- **表达式求值**：使用调度场算法解析法术表达式
- **类型系统**：严格的类型检查确保法术正确执行

### 2. 四维魔力体系
不同于传统的单一魔力概念，本模组采用四维魔力系统：
- **辐射系 (Radiation)**：能量释放和光效相关
- **温度系 (Temperature)**：热力学和温度控制
- **动量系 (Momentum)**：运动和冲击力相关
- **压力系 (Pressure)**：物理压力和压缩效应

### 3. 模块化设计
- **插件系统**：魔杖支持多种功能插件装配
- **组件化GUI**：现代响应式用户界面
- **可扩展架构**：易于添加新功能和内容

## 🎮 核心玩法

### 魔杖系统
```
[魔杖] → [法术序列] → [插件增强] → [魔法效果]
```

1. **法术编辑**：在魔杖界面中拖拽法术构建序列
2. **参数配置**：为法术设置具体的数值参数
3. **插件装配**：安装不同插件获得额外能力
4. **充能释放**：按住右键蓄力后释放魔法

### 法术分类示例
```
数字运算: 1 + 2 * 3
向量操作: position_of(player) + (0, 1, 0)
逻辑控制: if_true_do[ condition ]
循环结构: loop{ action }
实体交互: teleport(entity, vector)
```

## 🏗️ 技术架构亮点

### 现代化UI框架
- 自研的响应式GUI系统
- 平滑动画和过渡效果
- 插件化的界面组件

### 高性能执行引擎
- 法术序列实时编译
- 实体级别的魔法执行
- 智能的资源管理

### 网络同步机制
- 客户端/服务端数据同步
- 实时状态更新
- 高效的数据传输

## 🎨 视觉设计哲学

### 色彩编码系统
不同类型的法术使用不同颜色标识：
- 🔴 红色：实体相关法术
- 🟠 橙色：触发器法术  
- 🟡 黄色：流程控制
- 🟢 绿色：数值运算
- 🔵 蓝色：向量操作
- 🟣 紫色：布尔逻辑

### 界面交互原则
- **直观性**：操作符合直觉
- **反馈性**：及时的操作反馈
- **一致性**：统一的设计语言
- **可访问性**：支持不同技能水平的玩家

## 🚀 项目愿景

打造一个真正意义上的"可编程"游戏体验，让玩家：
- 学习基础的编程思维
- 享受创造的乐趣
- 体验逻辑之美
- 分享自己的魔法作品

## 📊 项目规模

- **代码行数**：~15,000行Java代码
- **法术种类**：60+种基础法术
- **插件类型**：10+种功能插件
- **GUI组件**：50+个自定义控件
